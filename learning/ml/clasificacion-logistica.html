Logística, SVM & KNN

De predecir números a predecir categorías. Tres algoritmos clásicos con geometrías muy distintas: fronteras probabilísticas, márgenes máximos y vecindades locales.

Área Machine Learning Duración ~35 min Prerrequisitos Módulo 05 — Regresión Lineal
Concepto

De regresión a clasificación

En regresión predecíamos un valor continuo $\hat{y} \in \mathbb{R}$. En clasificación la salida es discreta: ¿es un email spam o no? ¿Este tumor es benigno o maligno? ¿Qué dígito es esta imagen?

El caso más simple es la clasificación binaria: dos clases, que llamamos 0 y 1 (o $-1$ y $+1$ en SVM). Formalmente, buscamos una función $f: \mathbb{R}^d \to \{0,1\}$ que separe el espacio de features en dos regiones.

Frontera de decisión

La frontera de decisión (decision boundary) es la superficie que separa las regiones donde el modelo predice clase 0 de las que predice clase 1. Dependiendo del algoritmo, esta frontera puede ser lineal, curva, o una región compleja definida por ejemplos vecinos.

Los tres algoritmos de este módulo producen fronteras de decisión con filosofías completamente distintas — y entender esas diferencias es clave para saber cuándo usar cada uno.

Algoritmo I

Regresión Logística

A pesar del nombre, la regresión logística es un clasificador. La idea: tomar la salida lineal $z = \mathbf{w}^T\mathbf{x} + b$ y aplastarla al rango $[0,1]$ con la función sigmoide $\sigma$, interpretando el resultado como una probabilidad.

Función sigmoide y predicción
$$\sigma(z) = \frac{1}{1+e^{-z}} \qquad \hat{p} = \sigma(\mathbf{w}^T\mathbf{x}+b) \qquad \hat{y} = \begin{cases}1 & \hat{p} \geq 0.5 \\ 0 & \hat{p} < 0.5\end{cases}$$

La frontera de decisión $\hat{p}=0.5$ corresponde a $z=0$, es decir, al hiperplano $\mathbf{w}^T\mathbf{x}+b=0$ — es lineal.

En lugar del MSE, entrenamos minimizando la Binary Cross-Entropy (también llamada log-loss). Esta función mide cuán sorprendido estaría un modelo probabilístico con los datos observados:

Función de pérdida — Binary Cross-Entropy
$$\mathcal{L} = -\frac{1}{n}\sum_{i=1}^{n}\Big[y_i\log\hat{p}_i + (1-y_i)\log(1-\hat{p}_i)\Big]$$
¿Por qué no MSE para clasificación?

Con la sigmoide y MSE, la superficie de pérdida tiene muchos mínimos locales y los gradientes se aplanan cuando la predicción es muy incorrecta (el gradiente de $\sigma$ se acerca a 0). La cross-entropy tiene gradientes bien comportados en toda la superficie y es la derivación natural desde máxima verosimilitud.

Los gradientes resultan notablemente simples — derivando la cross-entropy sobre $z = \mathbf{w}^T\mathbf{x}+b$:

Gradientes de BCE respecto a w y b
$$\frac{\partial \mathcal{L}}{\partial \mathbf{w}} = \frac{1}{n}\sum_{i=1}^{n}(\hat{p}_i - y_i)\,\mathbf{x}_i \qquad \frac{\partial \mathcal{L}}{\partial b} = \frac{1}{n}\sum_{i=1}^{n}(\hat{p}_i - y_i)$$
logistic_regression.pyNumPy
import numpy as np

def sigmoid(z):
    return 1 / (1 + np.exp(-z))

def logistic_regression(X, y, lr=0.1, epochs=1000):
    n, d = X.shape
    w, b = np.zeros(d), 0.0

    for _ in range(epochs):
        z    = X @ w + b
        p    = sigmoid(z)               # probabilidades predichas
        err  = p - y                    # residuos

        dw = (1/n) * X.T @ err          # gradiente respecto a w
        db = (1/n) * err.sum()          # gradiente respecto a b

        w -= lr * dw
        b -= lr * db

    return w, b

Algoritmo II

Support Vector Machine (SVM)

La regresión logística encuentra alguna frontera que separa las clases. La SVM busca la mejor frontera: aquella que maximiza el margen, la distancia entre la frontera y los puntos más cercanos de cada clase.

Esos puntos más cercanos son los vectores de soporte — y son los únicos que importan para definir la frontera. Si el resto de los puntos desaparecieran, la frontera no cambiaría.

Problema de optimización (SVM lineal de margen duro)
$$\min_{\mathbf{w},b} \frac{1}{2}\|\mathbf{w}\|^2 \quad \text{sujeto a} \quad y_i(\mathbf{w}^T\mathbf{x}_i + b) \geq 1 \quad \forall i$$

El margen tiene ancho $\frac{2}{\|\mathbf{w}\|}$. Minimizar $\|\mathbf{w}\|^2$ equivale a maximizar el margen. En la práctica se usa la formulación de margen blando (soft margin), que permite algunas violaciones con penalidad $C$:

SVM de margen blando (hinge loss)
$$\mathcal{L} = \frac{1}{n}\sum_{i=1}^{n}\max(0,\, 1 - y_i(\mathbf{w}^T\mathbf{x}_i + b)) + \frac{\lambda}{2}\|\mathbf{w}\|^2$$
Kernel trick

El poder real de las SVM está en el kernel trick: si los datos no son linealmente separables, se proyectan implícitamente a un espacio de mayor dimensión mediante una función kernel $K(\mathbf{x}_i, \mathbf{x}_j)$, donde sí pueden serlo. Kernels comunes: lineal, polinomial $(\mathbf{x}_i^T\mathbf{x}_j + c)^d$, y RBF gaussiano $e^{-\gamma\|\mathbf{x}_i-\mathbf{x}_j\|^2}$.

Algoritmo III

K-Nearest Neighbors (KNN)

KNN es radicalmente distinto: no tiene parámetros que entrenar. En predicción, simplemente busca los $k$ ejemplos de entrenamiento más cercanos al punto de consulta y devuelve la clase mayoritaria entre ellos.

Predicción KNN
$$\hat{y} = \underset{c}{\operatorname{argmax}} \sum_{i \in \mathcal{N}_k(\mathbf{x})} \mathbf{1}[y_i = c]$$

$\mathcal{N}_k(\mathbf{x})$ denota los $k$ vecinos más cercanos de $\mathbf{x}$ según alguna métrica de distancia. La más común es la distancia Euclidiana, pero también se usan Manhattan ($L_1$), Minkowski ($L_p$), o coseno para texto.

Distancias de Minkowski
$$d_p(\mathbf{x}, \mathbf{z}) = \left(\sum_{j=1}^{d}|x_j - z_j|^p\right)^{1/p} \quad \begin{cases} p=1: & \text{Manhattan} \\ p=2: & \text{Euclídea} \\ p\to\infty: & \text{Chebyshev} \end{cases}$$
El parámetro k importa mucho

k=1: frontera muy irregular, sobreajuste — cada punto de entrenamiento define su propia región de Voronoi. k grande: frontera más suave pero puede subajustar. El valor óptimo se encuentra con validación cruzada. Siempre escalá los features antes de KNN: las distancias son sensibles a la escala.

knn.pyNumPy
import numpy as np
from collections import Counter

class KNN:
    def __init__(self, k=3):
        self.k = k

    def fit(self, X, y):           # sólo memorizamos
        self.X_train = X
        self.y_train = y

    def predict(self, X):
        return np.array([self._predict_one(x) for x in X])

    def _predict_one(self, x):
        dists = np.linalg.norm(self.X_train - x, axis=1)   # distancias euclidianas
        k_idx = np.argsort(dists)[:self.k]                  # índices de k vecinos
        k_labels = self.y_train[k_idx]
        return Counter(k_labels).most_common(1)[0][0]        # clase mayoritaria

¿Cuándo usar cada uno?

Algoritmo Frontera Output Entrenamiento Predicción Cuándo usarlo
Regresión Logística Lineal Probabilidad $\hat{p}$ $O(n \cdot d \cdot T)$ $O(d)$ Baseline rápido, interpretabilidad, texto con TF-IDF
SVM lineal Lineal (máximo margen) Score $\pm$ $O(n \cdot d \cdot T)$ $O(d)$ Texto, datos de alta dimensión, margen explícito
SVM RBF kernel No lineal Score $\pm$ $O(n^2)$–$O(n^3)$ $O(n_{sv} \cdot d)$ Datasets pequeños/medianos con estructura compleja
KNN No lineal (Voronoi) Clase (o prob.) $O(n)$ — trivial $O(n \cdot d)$ — lento Prototipos, detección de anomalías, datasets pequeños
Regla práctica

Siempre empezá con regresión logística como baseline. Si los datos no son linealmente separables y el dataset es pequeño, probá SVM con kernel RBF. KNN es útil como sanity check o cuando los datos tienen estructura de vecindad natural. Para datasets grandes (millones de ejemplos), GBM (Gradient Boosting) o redes neuronales suelen superar a SVM y KNN.

Evaluación

Más allá del accuracy

El accuracy (porcentaje de aciertos) es una métrica engañosa en datasets desbalanceados. Si el 95 % de los emails son legítimos, un clasificador que siempre predice "no spam" tiene 95 % de accuracy — y es completamente inútil.

Matriz de confusión y métricas derivadas
$$\text{Precision} = \frac{TP}{TP+FP} \qquad \text{Recall} = \frac{TP}{TP+FN} \qquad F_1 = 2\cdot\frac{P \cdot R}{P + R}$$
Pred. PositivoPred. Negativo
Real PositivoTP — Verdadero positivoFN — Falso negativo (Error tipo II)
Real NegativoFP — Falso positivo (Error tipo I)TN — Verdadero negativo
Precision vs Recall — el trade-off

Bajar el umbral de decisión (de 0.5 a 0.3) aumenta el Recall (detectamos más positivos reales) pero baja la Precision (más falsos positivos). La curva Precision-Recall y el área bajo ella (AUC-PR) son métricas más informativas que el accuracy para clases desbalanceadas.
