Regresión Lineal &
Gradiente Descendente

El primer paso del aprendizaje automático: ajustar una recta a datos y optimizar sus parámetros minimizando el error cuadrático con cálculo diferencial.

Área Machine Learning Duración ~30 min Prerrequisitos Derivadas básicas
Concepto

El problema: aprender de datos

Supongamos que tenemos una colección de pares $(x_i, y_i)$: horas de estudio y notas, metros cuadrados y precios de casas, temperatura y consumo eléctrico. La regresión lineal busca la recta que mejor explique esa relación.

Esa recta tiene la forma $\hat{y} = wx + b$, donde $w$ es la pendiente (weight) y $b$ el intercepto (bias). Aprender un modelo lineal significa encontrar los valores de $w$ y $b$ que minimicen el error entre las predicciones $\hat{y}_i$ y los valores reales $y_i$.

Idea central

El aprendizaje es optimización. Definimos qué significa "error", lo medimos con una función matemática (la pérdida), y usamos cálculo para reducirla sistemáticamente ajustando los parámetros del modelo.

Matemática

Error Cuadrático Medio (MSE)

Necesitamos una función que mida qué tan mala es la recta actual. El Error Cuadrático Medio suma los cuadrados de las diferencias entre predicciones y valores reales, dividido por el número de puntos $n$:

Función de pérdida
$$\mathcal{L}(w, b) = \frac{1}{n} \sum_{i=1}^{n} (\hat{y}_i - y_i)^2 = \frac{1}{n} \sum_{i=1}^{n} (wx_i + b - y_i)^2$$

¿Por qué elevar al cuadrado? Dos razones: los errores positivos y negativos no se cancelan entre sí, y penaliza más los errores grandes que los pequeños — una propiedad conveniente. También garantiza que $\mathcal{L}$ sea diferenciable y convexa en $w$ y $b$, lo que facilita la optimización.

Convexidad

La función $\mathcal{L}(w, b)$ tiene forma de paraboloide: un único mínimo global. Esto es una propiedad especial de la regresión lineal con MSE. En redes neuronales profundas la superficie de pérdida es mucho más compleja, con mínimos locales y puntos de silla.

  • 1Para cada punto $i$, calcula la predicción $\hat{y}_i = wx_i + b$

  • 1Calcula la predicción: $\hat{y}_i = wx_i + b$
  • 2Calcula el residuo: $e_i = \hat{y}_i - y_i$
  • 3Eleva al cuadrado: $e_i^2$
  • 4Promedia sobre todos los puntos: $\mathcal{L} = \frac{1}{n}\sum e_i^2$
Algoritmo

Gradiente Descendente

Tenemos una función de pérdida $\mathcal{L}(w, b)$. Queremos bajar "cuesta abajo" en esa superficie hasta llegar al mínimo. El gradiente $\nabla\mathcal{L}$ apunta en la dirección de máximo ascenso; si nos movemos en sentido contrario, bajamos.

Regla de actualización
$$w \leftarrow w - \alpha \frac{\partial \mathcal{L}}{\partial w} \qquad b \leftarrow b - \alpha \frac{\partial \mathcal{L}}{\partial b}$$

El hiperparámetro $\alpha$ (alpha) es el learning rate — controla el tamaño de cada paso. Las derivadas parciales se calculan por la regla de la cadena:

Derivadas parciales del MSE
$$\frac{\partial \mathcal{L}}{\partial w} = \frac{2}{n}\sum_{i=1}^{n}(wx_i + b - y_i)\cdot x_i \qquad \frac{\partial \mathcal{L}}{\partial b} = \frac{2}{n}\sum_{i=1}^{n}(wx_i + b - y_i)$$
El learning rate importa

Demasiado grande: el algoritmo oscila o diverge — salta por encima del mínimo repetidamente. Demasiado pequeño: converge muy lento, requiriendo miles de iteraciones. En la práctica se usa learning rate scheduling o adaptativo (Adam, RMSProp). En el módulo 12 se profundiza esto.

El algoritmo completo es notablemente simple. En Python con NumPy:

gradient_descent.pyNumPy
import numpy as np

def gradient_descent(X, y, lr=0.01, epochs=1000):
    n = len(X)
    w, b = 0.0, 0.0                     # inicialización en cero

    for epoch in range(epochs):
        y_hat = w * X + b               # predicciones actuales
        residuals = y_hat - y           # errores

        # gradientes
        dw = (2/n) * np.dot(residuals, X)
        db = (2/n) * np.sum(residuals)

        # actualización simultánea
        w -= lr * dw
        b -= lr * db

        if epoch % 100 == 0:
            mse = np.mean(residuals**2)
            print(f"Epoch {epoch:4d} | MSE: {mse:.4f} | w={w:.3f} b={b:.3f}")

    return w, b
Batch, Mini-batch y SGD

El código anterior es Batch Gradient Descent: usa todos los datos en cada paso. SGD (Stochastic) usa un solo punto aleatorio por iteración — es ruidoso pero más rápido para datasets grandes. Mini-batch (típicamente 32–256 ejemplos) es el estándar en deep learning: balance entre velocidad y estabilidad.

Teoría

Ecuación Normal: la solución exacta

Para regresión lineal existe una fórmula de forma cerrada. Derivar $\mathcal{L}$ respecto a los parámetros, igualando a cero y resolviendo, da la ecuación normal:

Solución analítica (forma vectorial)
$$\boldsymbol{\theta}^* = (\mathbf{X}^T\mathbf{X})^{-1}\mathbf{X}^T\mathbf{y}$$

Para el caso univariado ($\hat{y} = wx + b$) las fórmulas simplifican a:

Solución analítica (caso 1D)
$$w^* = \frac{\sum(x_i - \bar{x})(y_i - \bar{y})}{\sum(x_i - \bar{x})^2} \qquad b^* = \bar{y} - w^*\bar{x}$$

Si existe la solución exacta, ¿para qué usar gradiente descendente? La respuesta está en la escala.

MétodoComplejidadCuándo usarlo
Ecuación Normal$O(n \cdot d^2 + d^3)$ — invertir matriz $d{\times}d$Pocos features ($d \lesssim 10^4$), dataset pequeño
Gradiente Descendente (Batch)$O(n \cdot d)$ por iteraciónDatasets medianos, base del pipeline ML
SGD / Mini-batch$O(b \cdot d)$ por paso, $b \ll n$Millones de ejemplos, redes neuronales
Por qué GD es fundamental

Para redes neuronales profundas con millones de parámetros, invertir una matriz es computacionalmente imposible. El gradiente descendente — y sus variantes estocásticas — es el único método práctico. Entender regresión lineal con GD es el fundamento conceptual de todo el deep learning.

Práctica

Feature Scaling: por qué normalizar

Si los features tienen escalas muy distintas (por ejemplo, edad en [0,100] y precio en [0,1.000.000]), la superficie de pérdida se vuelve un elipsoide muy elongado. El gradiente "zigzaguea" por las paredes del elipsoide en lugar de ir directo al mínimo.

La solución es normalizar los features antes de entrenar. Dos técnicas comunes:

Normalización estándar (Z-score)
$$x' = \frac{x - \mu}{\sigma}$$
Normalización Min-Max
$$x' = \frac{x - x_{\min}}{x_{\max} - x_{\min}}$$
Regla práctica

Siempre escalá los features con los estadísticos del training set. Nunca con los del test set — eso constituye data leakage y produce métricas de evaluación engañosamente optimistas.

normalizacion.py
# Z-score normalization (nunca usar el test set para calcular mu/sigma)
mu_train  = X_train.mean(axis=0)
std_train = X_train.std(axis=0)

X_train_norm = (X_train - mu_train) / std_train
X_test_norm  = (X_test  - mu_train) / std_train   # ← mismos estadísticos
