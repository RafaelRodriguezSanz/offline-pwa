Motivación

¿Por qué la lógica proposicional no alcanza?

La lógica proposicional solo maneja hechos atómicos verdaderos o falsos: "Sócrates es mortal". No puede expresar relaciones generales como "todos los humanos son mortales" sin enumerar cada caso. La Lógica de Primer Orden (FOL, First-Order Logic) introduce variables, predicados y cuantificadores para hablar de objetos y sus relaciones de forma general.

Salto fundamental

Proposicional: mortal_socrates = true — hay que listar cada individuo.
Primer Orden: $\forall x \; \text{Humano}(x) \implies \text{Mortal}(x)$ — una sola regla cubre todos los individuos posibles.

Sintaxis

Los Componentes de FOL

Una fórmula en Lógica de Primer Orden se construye a partir de términos y fórmulas atómicas, conectadas con operadores lógicos y cuantificadores.

Gramática (simplificada)
$$\text{Término} ::= \text{constante} \mid \text{variable} \mid \text{función}(\text{término}_1, \ldots, \text{término}_n)$$ $$\text{Átomo} ::= \text{predicado}(\text{término}_1, \ldots, \text{término}_n)$$ $$\text{Fórmula} ::= \text{Átomo} \mid \lnot F \mid F_1 \land F_2 \mid F_1 \lor F_2 \mid F_1 \implies F_2 \mid \forall x\, F \mid \exists x\, F$$

PredicadoHumano(X)Propiedad o relación
Funciónpadre_de(X)Mapea objetos a objetos
∀ Universal∀x Humano(x) → Mortal(x)Para todo x
∃ Existencial∃x Filósofo(x) ∧ Ateniense(x)Existe al menos un x

Algoritmo central

Unificación — El Motor de FOL

La unificación es el proceso de encontrar una sustitución $\theta$ que haga que dos expresiones sean idénticas. Es el corazón del motor de inferencia: para aplicar una regla, hay que unificar su cabeza con el hecho en cuestión.

Definición de unificador
$$\text{UNIFICAR}(p, q) = \theta \quad \text{tal que} \quad p\theta = q\theta$$ $$\text{donde } \theta = \{V_1 / t_1,\; V_2 / t_2,\; \ldots\} \text{ es una sustitución de variables por términos}$$

El Unificador Más General (MGU) es la sustitución más simple posible — la que no compromete más variables de lo necesario.

Reglas de unificación (algoritmo de Robinson)

1. Constante = Constante: unifica solo si son idénticas.
2. Variable = Término: sustituir variable por término (si la variable no aparece en el término — occur check).
3. Función = Función: unificar nombre y cada argumento recursivamente.
4. Cualquier otra combinación: falla.

Ejemplos rápidos
Occur Check

El occur check impide sustituir una variable $X$ por un término que la contenga, como $X = f(X)$, lo que crearía una estructura infinita. Prolog estándar omite el occur check por eficiencia, lo que puede producir resultados incorrectos en casos patológicos. ISO Prolog provee unify_with_occurs_check/2 para el caso estricto.

Inferencia

Resolución — De Cláusulas a Pruebas

El principio de resolución de Robinson (1965) es una regla de inferencia que opera sobre cláusulas en Forma Normal Conjuntiva (CNF). Dadas dos cláusulas que contienen un literal y su negación, la resolución produce una nueva cláusula que las combina.

Regla de resolución
$$\frac{(L \lor \alpha) \quad (\lnot L \lor \beta)}{(\alpha \lor \beta)}$$ $$\text{donde } L \text{ es un literal y } \alpha, \beta \text{ son disyunciones de literales}$$

Para probar que una fórmula $\phi$ se deduce de una base de conocimiento $KB$, usamos refutación: agregamos $\lnot\phi$ a $KB$ y mostramos que el conjunto es insatisfacible (podemos derivar la cláusula vacía $\square$).

Prueba por refutación
$$KB \models \phi \iff KB \cup \{\lnot\phi\} \text{ es insatisfacible}$$ $$KB \cup \{\lnot\phi\} \vdash_{Res} \square \implies KB \models \phi$$

Ejemplo paso a paso: queremos probar que Sócrates es mortal.

Árbol de resolución
resolución manual
// Cláusulas iniciales (KB + negación de la meta):
C1: ¬Humano(X)  ∨  Mortal(X)      // "todo humano es mortal"
C2:  Humano(sócrates)              // hecho
C3: ¬Mortal(sócrates)             // negación de la meta

// Paso 1: resolver C1 y C2 con θ = {X/sócrates}
C4: Mortal(sócrates)              // resolvente

// Paso 2: resolver C4 y C3
C5: □                             // cláusula vacía → refutación exitosa

// Conclusión: KB ⊨ Mortal(sócrates)  ✓

Cláusulas de Horn

Prolog trabaja con un subconjunto de FOL llamado cláusulas de Horn: cláusulas con a lo sumo un literal positivo. La forma cabeza :- cuerpo es una cláusula de Horn. Este subconjunto es decidible y el backward chaining sobre él es completo para refutación — eso es lo que hace Prolog eficiente.

Transformación

Forma Normal Conjuntiva (CNF)

Para aplicar resolución, toda fórmula debe estar en CNF: una conjunción de disyunciones de literales. Cualquier fórmula FOL puede convertirse a CNF en siete pasos.

Pipeline de conversión a CNF
$$\underbrace{\forall x\; [A(x) \implies B(x)]}_{\text{1. Original}} \xrightarrow{\text{elim. } \implies} \underbrace{\forall x\; [\lnot A(x) \lor B(x)]}_{\text{2. Sin implicaciones}} \xrightarrow{\text{mover } \lnot} \underbrace{\forall x\; [\lnot A(x) \lor B(x)]}_{\text{3. NNF}}$$ $$\xrightarrow{\text{Skolem}} \underbrace{\lnot A(x) \lor B(x)}_{\text{4. Sin }\exists} \xrightarrow{\text{dist. } \lor/\land} \underbrace{(\lnot A(x) \lor B(x))}_{\text{5. CNF}}$$

La skolemización elimina los cuantificadores existenciales reemplazando $\exists y$ con una función de Skolem $f(x_1,\ldots,x_n)$ donde $x_i$ son las variables universales en cuyo alcance aparece $y$.

Conexión con LLMs

Aunque los LLMs modernos no usan FOL explícita, la idea de representar conocimiento en embeddings y razonar sobre él tiene raíces directas en FOL. Los Neural Theorem Provers combinan redes neuronales con unificación suave. Y el chain-of-thought prompting es, en esencia, inducir al modelo a simular un proceso de inferencia paso a paso — el mismo ideal de GOFAI, pero aprendido en lugar de codificado.

Completitud y Limitaciones

El procedimiento de resolución es refutación-completo: si $KB \models \phi$, eventualmente lo demostrará. Pero FOL completa es solo semidecidible: si $KB \not\models \phi$, el procedimiento puede no terminar nunca.

Teorema de Church (1936) & Turing (1936)
$$\text{El problema de validez en FOL es} \textbf{ indecidible}$$ $$\nexists \text{ algoritmo que termine siempre y decida si } KB \models \phi$$

Esta limitación teórica, combinada con la explosión combinatoria en espacios de búsqueda grandes, fue la segunda razón por la que GOFAI cedió terreno a los métodos estadísticos.
