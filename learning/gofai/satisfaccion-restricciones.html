Definición

¿Qué es un CSP?

Un problema de satisfacción de restricciones se define por tres elementos: variables, dominios y restricciones. El objetivo es asignar un valor del dominio a cada variable de forma que todas las restricciones se cumplan simultáneamente.

Definición formal
$$\text{CSP} = \langle X,\, D,\, C \rangle$$ $$X = \{X_1, X_2, \ldots, X_n\} \quad \text{(variables)}$$ $$D = \{D_1, D_2, \ldots, D_n\} \quad \text{(dominios, } D_i \text{ son los valores posibles de } X_i\text{)}$$ $$C = \{C_1, C_2, \ldots, C_m\} \quad \text{(restricciones sobre subconjuntos de } X\text{)}$$

Una solución es una asignación completa y consistente: cada variable tiene un valor, y ninguna restricción es violada. Una asignación parcial es consistente si las restricciones sobre las variables ya asignadas se cumplen.

Por qué CSP es poderoso

A diferencia de la búsqueda ciega, un CSP expone la estructura interna del problema. Esto permite usar propagación de restricciones para eliminar valores imposibles antes de asignarlos, reduciendo dramáticamente el espacio de búsqueda.

Algoritmo base

Backtracking

El algoritmo de backtracking para CSP es una búsqueda DFS que asigna valores uno a uno. Cuando una asignación parcial viola una restricción, retrocede y prueba el siguiente valor. Con CSPs, la estructura nos permite podar el árbol de búsqueda antes de explorar ramas enteras.

Árbol de búsqueda para N=4 (columnas ↓, filas →)
$$\underbrace{Q_1=1}_{\text{columna 1}} \to \underbrace{Q_2=3}_{\text{col. 2}} \to \underbrace{Q_3=?}_{\text{col. 3, sin valores}} \to \text{backtrack}$$ $$Q_1=2 \to Q_2=4 \to Q_3=1 \to Q_4=3 \quad \checkmark$$
backtracking_csp.py
def backtrack(assignment, csp):
    if is_complete(assignment, csp):
        return assignment                    # solución encontrada

    var = select_unassigned_variable(assignment, csp)  # heurística MRV

    for value in order_domain_values(var, assignment, csp):  # heurística LCV
        if is_consistent(var, value, assignment, csp):
            assignment[var] = value

            # Forward Checking: propagar y verificar dominios
            inferences = inference(csp, var, value)
            if inferences is not None:
                result = backtrack(assignment, csp)
                if result: return result

            # Backtrack: deshacer asignación e inferencias
            del assignment[var]
            restore_domains(csp, inferences)

    return None                              # falla → backtrack al padre

Heurísticas de Selección

El orden en que se eligen variables y valores impacta enormemente el rendimiento:

MRV — Minimum Remaining Values
Elegir la variable con el dominio más pequeño. Falla primero, ahorra trabajo. También llamado fail-first heuristic.
$\text{var}^* = \arg\min_{v \notin \text{asig}} |D_v|$
LCV — Least Constraining Value
Elegir el valor que elimina menos opciones de las variables vecinas. Deja más flexibilidad para el futuro.
$\text{val}^* = \arg\max_v \sum_{j \in \text{vecinos}} |D_j \setminus \{v\}|$
Degree Heuristic
Elegir la variable con más restricciones sobre variables no asignadas. Rompe empates de MRV.
$\text{var}^* = \arg\max_{v \notin \text{asig}} \deg(v)$
Forward Checking
Al asignar $X_i = v$, eliminar $v$ de los dominios de las variables vecinas. Detecta fallos temprano.
$D_j \leftarrow D_j \setminus \{v\} \quad \forall j \in \text{vecinos}(X_i)$
Propagación de arcos

AC-3 — Arc Consistency

Un arco $(X_i, X_j)$ es arco-consistente si para todo valor en $D_i$ existe al menos un valor en $D_j$ compatible con la restricción entre $X_i$ y $X_j$. El algoritmo AC-3 fuerza arc-consistency en todas las variables antes y durante la búsqueda.

Condición de arc-consistency
$$\forall\, x \in D_i,\; \exists\, y \in D_j : \langle X_i = x,\, X_j = y \rangle \text{ satisface } C_{ij}$$
ac3.py
from collections import deque

def ac3(csp):
    queue = deque(csp.all_arcs())      # todos los arcos (Xi, Xj)

    while queue:
        Xi, Xj = queue.popleft()

        if revise(csp, Xi, Xj):            # ¿se eliminó algún valor de Di?
            if len(csp.domains[Xi]) == 0:
                return False                   # dominio vacío → sin solución
            for Xk in csp.neighbors(Xi) - {Xj}:
                queue.append((Xk, Xi))  # re-encolar vecinos afectados

    return True

def revise(csp, Xi, Xj):
    revised = False
    for x in list(csp.domains[Xi]):
        # ¿existe algún y en Dj que satisfaga la restricción?
        if not any(csp.satisfies(x, y, Xi, Xj) for y in csp.domains[Xj]):
            csp.domains[Xi].remove(x)
            revised = True
    return revised
Complejidad de AC-3
$$O(e \cdot d^3) \quad \text{donde } e = \text{arcos},\; d = \max|D_i|$$

Conexión con Machine Learning

Los CSPs son deterministas: los dominios son discretos y las restricciones son duras (se cumplen o no). ML maneja el mundo real con restricciones blandas y incertidumbre. Las Redes Bayesianas (un próximo módulo) son la versión probabilística de los CSPs: variables con dominios, pero las restricciones se convierten en distribuciones de probabilidad condicional.
