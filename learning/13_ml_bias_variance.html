Bias–Variance, Overfitting &
Regularización

El delicado equilibrio del Machine Learning. Entendé por qué los modelos muy simples fallan, por qué los muy complejos memorizan, y cómo usar L1 y L2 para domesticarlos.

Área Machine Learning Duración ~25 min Prerrequisitos Regresión Lineal
Concepto

El dilema del aprendizaje

El objetivo central del Machine Learning no es lograr que el modelo tenga cero error en los datos de entrenamiento (eso es trivial, basta con memorizar). El objetivo es que el modelo generalice, es decir, que tenga un error bajo en datos nuevos que nunca antes ha visto.

Escenario Complejidad Error en Training Error en Test Síntoma
Underfitting (Subajuste) Muy simple (ej: línea recta para curva) Alto Alto Incapaz de capturar el patrón subyacente.
Good Fit (Óptimo) Adecuada Bajo Bajo Captura la señal, ignora el ruido.
Overfitting (Sobreajuste) Muy complejo (ej: polinomio grado 15) Casi cero Muy alto Memoriza el ruido y fluctúa salvajemente.
Teoría

El Trade-off Bias/Varianza

Matemáticamente, el error de predicción esperado de un modelo se puede descomponer en tres partes fundamentales:

Descomposición del Error Expected
$$Error = \text{Bias}^2 + \text{Varianza} + \text{Ruido Irreducible}$$
  • Bias (Sesgo): Error introducido por aproximar un problema de la vida real mediante un modelo demasiado simple. Genera underfitting.
  • Varianza: Sensibilidad del modelo a pequeñas fluctuaciones en el set de entrenamiento. Un modelo con alta varianza cambia drásticamente si se entrena con un set de datos ligeramente distinto. Genera overfitting.
  • Ruido Irreducible: La variabilidad inherente de los datos ($\epsilon$) que ningún modelo puede predecir.
El Trade-off

A medida que aumentamos la complejidad del modelo (por ejemplo, agregando más variables o subiendo el grado de un polinomio), el Bias disminuye pero la Varianza aumenta. El arte del Machine Learning es encontrar el punto exacto ("sweet spot") que minimiza el error total.

Algoritmo

Domesticando modelos: L1 y L2

Cuando tenemos features muy complejas o correlacionadas, los pesos (coeficientes) del modelo pueden volverse astronómicamente grandes en su intento por ajustar el ruido. La Regularización consiste en añadir una "multa" a la función de pérdida por tener pesos grandes.

Loss Regularizada
$$\mathcal{L} = \text{MSE}(y, \hat{y}) + \lambda \cdot \text{Penalidad}(W)$$
Tipo Nombre Penalidad Efecto Principal
L2 Ridge Regression $\sum w_i^2$ Reduce todos los pesos uniformemente. Evita valores extremos. Mantiene todos los features.
L1 Lasso Regression $\sum |w_i|$ Empuja algunos pesos exactamente a cero. Actúa como un selector de características natural.
Hiperparámetro Lambda ($\lambda$)

$\lambda$ controla la fuerza de la regularización. Si $\lambda=0$, volvemos a OLS normal (peligro de overfitting). Si $\lambda \to \infty$, todos los pesos colapsan a $0$ y el modelo predice una línea plana (underfitting). Debemos buscar un $\lambda$ intermedio mediante Validación Cruzada.

Metodología

Validación Cruzada (K-Fold)

¿Cómo elegimos el mejor grado del polinomio o el mejor $\lambda$ si no podemos tocar el set de Test? Dividimos nuestro set de entrenamiento en $K$ partes (folds). Entrenamos en $K-1$ partes y validamos en la parte restante. Repetimos $K$ veces, rotando la parte de validación.

Train (Folds 1-4)
Val (Fold 5)
Train (Folds 1-3)
Val (Fold 4)
Train (Fold 5)
... repite K veces ...

El rendimiento del hiperparámetro es el promedio del error en los $K$ folds de validación. Una vez encontrado el mejor $\lambda$, reentrenamos el modelo final usando todo el set de entrenamiento.
