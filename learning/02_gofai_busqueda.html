Fundamento

El Espacio de Estados

Un problema de búsqueda se define formalmente con cuatro componentes: un estado inicial, un conjunto de estados meta, una función de transición que describe los movimientos posibles, y un costo por transición.

Definición formal

Un problema de búsqueda es una tupla $\langle S, s_0, A, T, G, c \rangle$ donde $S$ es el espacio de estados, $s_0$ el estado inicial, $A$ las acciones, $T: S \times A \to S$ la función de transición, $G \subseteq S$ los estados meta, y $c: A \to \mathbb{R}^+$ el costo de cada acción.

Una solución es una secuencia de acciones $a_1, a_2, \ldots, a_n$ que lleva de $s_0$ a algún $g \in G$. Los algoritmos de búsqueda difieren en qué nodo expanden a continuación — esa decisión define su comportamiento completo.

Costo de camino
$$g(n) = \sum_{i=1}^{d} c(a_i) \qquad \text{donde } d = \text{profundidad del nodo } n$$

En un laberinto, cada celda es un estado, las acciones son moverse en 4 direcciones, el costo es 1 por paso, y el objetivo es llegar a la celda de destino.

BFS — Búsqueda por Anchura

BFS expande primero todos los nodos a profundidad 1, luego a profundidad 2, y así sucesivamente. Usa una cola FIFO. Garantiza encontrar el camino con menor cantidad de pasos (óptimo cuando todos los costos son iguales).

Invariante de BFS
$$\forall n \in \text{frontera}: \; d(n) \geq d(\text{nodo expandido más reciente})$$
bfs.py
from collections import deque

def bfs(grid, start, goal):
    queue   = deque([start])      # FIFO — el orden importa
    visited = {start: None}       # nodo → padre (para reconstruir camino)

    while queue:
        node = queue.popleft()    # siempre el más antiguo
        if node == goal:
            return reconstruct(visited, goal)

        for neighbor in get_neighbors(grid, node):
            if neighbor not in visited:
                visited[neighbor] = node
                queue.append(neighbor)

    return None  # sin solución

DFS — Búsqueda por Profundidad

DFS explora tan profundo como sea posible antes de retroceder (backtrack). Usa una pila LIFO. Muy eficiente en memoria — $O(bm)$ vs $O(b^d)$ de BFS — pero no garantiza encontrar el camino más corto y puede perderse en ramas infinitas.

Cuidado

DFS sin detección de ciclos puede entrar en un bucle infinito en grafos con ciclos. Siempre hay que mantener el conjunto de nodos visitados.

dfs.py
def dfs(grid, start, goal):
    stack   = [start]             # LIFO — usamos lista como pila
    visited = {start: None}

    while stack:
        node = stack.pop()        # siempre el más reciente
        if node == goal:
            return reconstruct(visited, goal)

        for neighbor in get_neighbors(grid, node):
            if neighbor not in visited:
                visited[neighbor] = node
                stack.append(neighbor)   # se explora ANTES que los hermanos

    return None

A* — Búsqueda Informada

A* combina el costo real del camino recorrido $g(n)$ con una heurística $h(n)$ que estima el costo restante hasta la meta. Expande siempre el nodo con menor $f(n) = g(n) + h(n)$.

Función de evaluación de A*
$$f(n) = g(n) + h(n)$$ $$g(n) = \text{costo real desde el inicio hasta } n$$ $$h(n) = \text{estimación del costo desde } n \text{ hasta la meta}$$

Para que A* sea óptimo, la heurística debe ser admisible: nunca sobreestima el costo real.

Condición de admisibilidad
$$h(n) \leq h^*(n) \quad \forall n$$ $$\text{donde } h^*(n) \text{ es el costo real óptimo desde } n \text{ hasta la meta}$$

En un laberinto, una heurística admisible clásica es la distancia de Manhattan: no puede haber un camino más corto que la distancia en línea recta (por celdas).

Distancia de Manhattan como heurística
$$h(n) = |x_n - x_{\text{meta}}| + |y_n - y_{\text{meta}}|$$
Consistencia (heurística más fuerte)

Una heurística es consistente (o monótona) si $h(n) \leq c(n, a, n') + h(n')$ para todo sucesor $n'$ de $n$. Consistencia implica admisibilidad y garantiza que A* nunca revisita un nodo expandido — más eficiente en la práctica.

astar.py
import heapq

def heuristic(a, b):                    # Manhattan distance
    return abs(a[0]-b[0]) + abs(a[1]-b[1])

def astar(grid, start, goal):
    heap    = [(0, start)]              # (f, nodo) — min-heap por f
    g_cost  = {start: 0}
    parent  = {start: None}

    while heap:
        f, node = heapq.heappop(heap)

        if node == goal:
            return reconstruct(parent, goal)

        for neighbor in get_neighbors(grid, node):
            new_g = g_cost[node] + 1    # costo de paso = 1
            if neighbor not in g_cost or new_g < g_cost[neighbor]:
                g_cost[neighbor] = new_g
                f_val = new_g + heuristic(neighbor, goal)
                heapq.heappush(heap, (f_val, neighbor))
                parent[neighbor] = node

    return None

Conexión con el resto del curso

Estos algoritmos son la base de muchos sistemas modernos: GPS y mapas usan A* o Dijkstra, juegos de tablero (ajedrez, Go) combinan búsqueda con evaluación heurística, y los planificadores de IA usan variantes de BFS. Incluso el entrenamiento de modelos de ML puede verse como búsqueda en el espacio de parámetros — aunque el espacio es de millones de dimensiones.

Más allá: Búsqueda en Espacios Continuos

En problemas reales el espacio de estados no es un laberinto discreto. Los planificadores de movimiento para robots usan RRT (Rapidly-exploring Random Trees), que muestrea el espacio aleatoriamente. Los juegos de tablero usan MCTS (Monte Carlo Tree Search), que combina búsqueda con simulaciones aleatorias para estimar el valor de cada nodo.

Y cuando el espacio de búsqueda no tiene estructura explotable por una heurística — o cuando la función de costo no es diferenciable — entran en juego los Algoritmos Genéticos, que veremos en el Área 6.
